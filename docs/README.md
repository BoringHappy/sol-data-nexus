# docs/

Project documentation.

- [`architecture.md`](architecture.md) — canonical system diagram, component map, data-flow narrative, and alternatives considered.
- [`adr/`](adr/) — Architecture Decision Records in the form **Context / Decision / Consequences**. Read these to understand *why* the project looks the way it does.
- `runbook.md` (added by task [#15](https://github.com/BoringHappy/sol-data-nexus/issues/15)) — reproduction guide. From a clean machine to a green smoke test in ten minutes.
- `results/` (later) — metric backfill snapshots committed for reviewer visibility.
- `demo/` (later) — screen-recording and assets for the end-to-end demo.

Metric specifications live in [`../specs/`](../specs/), not here, because they evolve under their own versioning policy.
